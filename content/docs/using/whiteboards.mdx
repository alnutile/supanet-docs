A **whiteboard** is a canvas you and your team can draw on together to plan, sketch, and diagram. SupaNet uses Excalidraw, so you get a familiar drawing experience with shapes, arrows, text, and more — all saved and synced in real time.

## Why whiteboards are useful

Whiteboards give you a space to think visually. You can:

- **Sketch and diagram** — draw flowcharts, mind maps, wireframes, and system diagrams.
- **Collaborate in real time** — multiple people can draw on the same board at once and see each other's cursors and changes as they happen.
- **Have the assistant draw** — ask Claude to create a diagram, and it updates the whiteboard in place.
- **Keep your ideas organized** — save whiteboards alongside your artifacts and to-dos in collections so the team can find them.
- **Persist your thinking** — changes are saved automatically, so your work doesn't get lost.

## Where to find whiteboards

On the sidebar, find **Whiteboards** under **Assets**. You'll see a list of all your whiteboards.

## Creating a whiteboard

Click **New whiteboard** to create one. You can give it a name and start drawing right away. The canvas is empty at first — you decide what goes on it.

If you're opening a whiteboard someone else created, you can see their cursor moving in real time as they draw.

## Drawing on a whiteboard

Use Excalidraw's tools:

- **Shapes** — rectangles, circles, diamonds, and more. Click a shape tool and drag on the canvas.
- **Arrows and lines** — connect shapes to show flow or relationships.
- **Text** — click the text tool and type labels, notes, or descriptions.
- **Freedraw** — sketch freehand with the pen tool.
- **Edit** — select any element and drag it to move it, or resize it by dragging its corners.

The toolbar is at the top of the canvas. Hover over each tool to see what it does.

## Real-time collaboration

When a whiteboard is open:

- You see other team members' **cursors** (labeled with their names) as they move.
- Changes are **synced instantly** — when someone adds a shape or moves it, you see it right away.
- Your work is **saved automatically** as you draw, so there's no "save" button to click.
- If someone else updates the board from another device or via the assistant, the changes appear on your canvas.

## Sharing and visibility

Each whiteboard has a visibility setting:

- **Private** — only you (and admins) can see and edit it.
- **Workspace** — everyone on your team can see it and draw on it together.

Toggle visibility in the whiteboard editor. Team members can only edit a whiteboard if you've shared it as workspace-visible.

## Collections and context

You can file whiteboards into **collections** — the same named groups you use for artifacts, files, and to-dos. From the Whiteboards list page, select whiteboards and click **Add to collection** to file them.

When you use a collection in Chat (via the 📚 picker), the assistant can see the whiteboards in that collection and help you edit them or create new ones. The assistant can also create a whiteboard and file it into a collection as part of a larger task.

## Chat with a whiteboard

When you have a whiteboard open, you can:

- Click **Chat** to open a conversation about the board.
- Tell the assistant to update it — "add a box for this" or "draw a timeline".
- The assistant reads the board's current state and makes changes in place instead of creating a new one.

## Automation and the API

The assistant and MCP can also manage whiteboards: they can create them, list them, read the current state, and make edits — all without you touching the UI. This means an agent can generate a diagram on a schedule, and your team always has the latest version.
